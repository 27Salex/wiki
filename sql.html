SQL General

Structured Query Language — Estándar ANSI/ISO

¿Qué es SQL?

SQL (Structured Query Language) es el lenguaje estándar para gestionar y manipular bases de datos relacionales. Fue desarrollado en los años 70 por IBM y estandarizado por ANSI e ISO. Es utilizado por prácticamente todos los sistemas de gestión de bases de datos relacionales (RDBMS): MySQL, PostgreSQL, SQL Server, Oracle, SQLite, etc.

SQL se divide en varios sublenguajes según el tipo de operación:

  • DDL (Data Definition Language): CREATE, ALTER, DROP, TRUNCATE — definen la estructura.
  • DML (Data Manipulation Language): SELECT, INSERT, UPDATE, DELETE — manipulan datos.
  • DCL (Data Control Language): GRANT, REVOKE — controlan permisos.
  • TCL (Transaction Control Language): COMMIT, ROLLBACK, SAVEPOINT — controlan transacciones.

DDL — Definición de Datos

CREATE TABLE

Crea una nueva tabla con columnas y restricciones (constraints).

-- Crear tabla de empleados con restricciones
CREATE TABLE empleados (
    id          INT PRIMARY KEY,
    nombre      VARCHAR(100) NOT NULL,
    email       VARCHAR(150) UNIQUE,
    salario     DECIMAL(10,2) DEFAULT 0.00,
    depto_id    INT,
    fecha_alta  DATE NOT NULL,
    CONSTRAINT fk_depto FOREIGN KEY (depto_id)
        REFERENCES departamentos(id)
        ON DELETE SET NULL
        ON UPDATE CASCADE
);

Restricciones Avanzadas (CHECK y Reglas de Propagación)

Las restricciones aseguran la integridad y consistencia de los datos en la base de datos.

-- Ejemplo con CONSTRAINT CHECK y reglas de propagación de claves foráneas
CREATE TABLE series_episodios (
    id_episodio   INT PRIMARY KEY,
    titulo        VARCHAR(150) NOT NULL,
    temporada     INT NOT NULL,
    depto_id      INT,
    
    -- CONSTRAINT CHECK: valida que la temporada esté entre 1 y 10
    CONSTRAINT chk_temporada_rango CHECK (temporada BETWEEN 1 AND 10),
    
    -- ON UPDATE CASCADE: Si cambia la PK en departamentos, se actualiza automáticamente aquí.
    -- ON DELETE CASCADE: Si se borra el departamento, se borran automáticamente los episodios asociados.
    CONSTRAINT fk_episodio_depto FOREIGN KEY (depto_id)
        REFERENCES departamentos(id)
        ON UPDATE CASCADE
        ON DELETE CASCADE
);

-- Clave Foránea con restricción RESTRICT:
-- Impide borrar o modificar la PK referenciada si existen registros asociados en la tabla hija.
CREATE TABLE personajes (
    id_personaje  INT PRIMARY KEY,
    nombre        VARCHAR(100) NOT NULL,
    depto_id      INT,
    
    CONSTRAINT fk_personaje_depto FOREIGN KEY (depto_id)
        REFERENCES departamentos(id)
        ON DELETE RESTRICT
        ON UPDATE RESTRICT
);

ALTER TABLE

Modifica la estructura de una tabla existente.

-- Añadir columna
ALTER TABLE empleados ADD telefono VARCHAR(20);

-- Modificar tipo de columna
ALTER TABLE empleados ALTER COLUMN nombre VARCHAR(200);

-- Eliminar columna
ALTER TABLE empleados DROP COLUMN telefono;

-- Añadir restricción CHECK
ALTER TABLE empleados
ADD CONSTRAINT chk_salario CHECK (salario >= 0);

DROP y TRUNCATE

-- DROP elimina la tabla y su estructura por completo
DROP TABLE IF EXISTS empleados;

-- TRUNCATE vacía todos los datos pero mantiene la estructura
TRUNCATE TABLE empleados;

DML — Manipulación de Datos

SELECT — Consultas

La sentencia más usada en SQL. Permite recuperar datos de una o más tablas.

-- Consulta básica con alias y filtro
SELECT
    e.nombre        AS "Empleado",
    d.nombre        AS "Departamento",
    e.salario
FROM empleados e
JOIN departamentos d ON e.depto_id = d.id
WHERE e.salario > 30000
ORDER BY e.salario DESC;

INSERT

-- Insertar una fila
INSERT INTO empleados (id, nombre, email, salario, depto_id, fecha_alta)
VALUES (1, 'Ana García', 'ana@empresa.com', 35000, 2, '2025-01-15');

-- Insertar múltiples filas
INSERT INTO empleados (id, nombre, email, salario, depto_id, fecha_alta)
VALUES
    (2, 'Carlos López',  'carlos@empresa.com', 42000, 1, '2024-06-01'),
    (3, 'María Ruiz',    'maria@empresa.com',  38000, 3, '2025-03-10');

UPDATE

-- Actualizar salario de un empleado
UPDATE empleados
SET salario = salario * 1.10
WHERE depto_id = 2;

-- Actualizar con subconsulta
UPDATE empleados
SET depto_id = (SELECT id FROM departamentos WHERE nombre = 'IT')
WHERE email = 'ana@empresa.com';

-- Plantilla de actualización compleja utilizando UPDATE con INNER JOIN (Estándar / T-SQL / PostgreSQL)
-- Modifica salarios basándose en filtros aplicados a la tabla de departamentos vinculada
UPDATE e
SET e.salario = e.salario * 1.10
FROM empleados e
INNER JOIN departamentos d ON e.depto_id = d.id
WHERE d.nombre = 'IT';

-- Plantilla de actualización compleja con INNER JOIN en MySQL (sintaxis alternativa)
UPDATE empleados e
INNER JOIN departamentos d ON e.depto_id = d.id
SET e.salario = e.salario * 1.10
WHERE d.nombre = 'Ventas';

DELETE

-- Eliminar registros con condición
DELETE FROM empleados
WHERE fecha_alta < '2020-01-01';

-- ⚠️ Sin WHERE elimina TODOS los registros
DELETE FROM empleados;

JOINs — Combinación de Tablas

Los JOINs permiten combinar filas de dos o más tablas basándose en una columna relacionada.

INNER JOIN

Devuelve solo las filas que tienen coincidencias en ambas tablas.

-- Empleados CON departamento asignado
SELECT e.nombre, d.nombre AS departamento
FROM empleados e
INNER JOIN departamentos d ON e.depto_id = d.id;

LEFT JOIN

Devuelve todas las filas de la tabla izquierda y las coincidencias de la derecha. Si no hay coincidencia, muestra NULL.

-- TODOS los empleados, aunque no tengan departamento
SELECT e.nombre, COALESCE(d.nombre, 'Sin asignar') AS departamento
FROM empleados e
LEFT JOIN departamentos d ON e.depto_id = d.id;

RIGHT JOIN

Inverso al LEFT JOIN: devuelve todas las filas de la tabla derecha.

-- TODOS los departamentos, aunque no tengan empleados
SELECT d.nombre AS departamento, COUNT(e.id) AS total
FROM empleados e
RIGHT JOIN departamentos d ON e.depto_id = d.id
GROUP BY d.nombre;

FULL OUTER JOIN

Combina LEFT y RIGHT: devuelve todas las filas de ambas tablas.

SELECT e.nombre, d.nombre AS departamento
FROM empleados e
FULL OUTER JOIN departamentos d ON e.depto_id = d.id;

CROSS JOIN

Producto cartesiano: combina cada fila de la primera tabla con todas las de la segunda.

-- Útil para generar combinaciones
SELECT t.turno, d.nombre
FROM turnos t
CROSS JOIN departamentos d;

Funciones Agregadas y GROUP BY

-- COUNT, SUM, AVG, MIN, MAX
SELECT
    d.nombre                     AS departamento,
    COUNT(e.id)                 AS total_empleados,
    AVG(e.salario)              AS salario_medio,
    MAX(e.salario)              AS salario_max,
    MIN(e.salario)              AS salario_min,
    SUM(e.salario)              AS masa_salarial
FROM empleados e
JOIN departamentos d ON e.depto_id = d.id
GROUP BY d.nombre
HAVING AVG(e.salario) > 30000
ORDER BY salario_medio DESC;

Subconsultas (Subqueries)

-- Subconsulta escalar en WHERE
SELECT nombre, salario
FROM empleados
WHERE salario > (SELECT AVG(salario) FROM empleados);

-- Subconsulta con IN
SELECT nombre
FROM empleados
WHERE depto_id IN (
    SELECT id FROM departamentos WHERE nombre LIKE '%Ventas%'
);

-- Subconsulta correlacionada con EXISTS
SELECT d.nombre
FROM departamentos d
WHERE EXISTS (
    SELECT 1 FROM empleados e
    WHERE e.depto_id = d.id AND e.salario > 50000
);

CTEs (Common Table Expressions)

Las CTEs permiten crear consultas temporales con nombre que mejoran la legibilidad.

-- CTE para calcular ranking de salarios
WITH ranking_salarios AS (
    SELECT
        nombre,
        salario,
        depto_id,
        RANK() OVER (ORDER BY salario DESC) AS posicion
    FROM empleados
)
SELECT r.nombre, r.salario, r.posicion, d.nombre AS depto
FROM ranking_salarios r
JOIN departamentos d ON r.depto_id = d.id
WHERE r.posicion <= 5;

Window Functions

Realizan cálculos sobre un conjunto de filas relacionadas con la fila actual, sin colapsar los resultados como GROUP BY.

SELECT
    nombre,
    depto_id,
    salario,
    -- Promedio por departamento sin agrupar
    AVG(salario) OVER (PARTITION BY depto_id) AS avg_depto,
    -- Número de fila por departamento
    ROW_NUMBER() OVER (
        PARTITION BY depto_id ORDER BY salario DESC
    ) AS fila,
    -- Acumulado de salario
    SUM(salario) OVER (
        ORDER BY fecha_alta
        ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
    ) AS salario_acumulado
FROM empleados;

Vistas e Índices

Vistas (Views)

Una vista es una consulta almacenada que se comporta como una tabla virtual.

-- Crear vista
CREATE VIEW v_resumen_departamento AS
SELECT
    d.nombre AS departamento,
    COUNT(e.id) AS empleados,
    ROUND(AVG(e.salario), 2) AS salario_medio
FROM departamentos d
LEFT JOIN empleados e ON d.id = e.depto_id
GROUP BY d.nombre;

-- Usar la vista como una tabla
SELECT * FROM v_resumen_departamento
WHERE empleados > 5;

Índices

Los índices aceleran las consultas a cambio de más espacio en disco y escrituras más lentas.

-- Índice simple
CREATE INDEX idx_emp_nombre ON empleados(nombre);

-- Índice compuesto (útil para búsquedas combinadas)
CREATE INDEX idx_emp_depto_sal ON empleados(depto_id, salario);

-- Índice único (impide duplicados)
CREATE UNIQUE INDEX idx_emp_email ON empleados(email);

TCL — Control de Transacciones

Las transacciones aseguran que un grupo de operaciones se ejecute de forma atómica (todo o nada).

BEGIN TRANSACTION;

UPDATE cuentas SET saldo = saldo - 500 WHERE id = 1;
UPDATE cuentas SET saldo = saldo + 500 WHERE id = 2;

-- Si todo va bien:
COMMIT;

-- Si hay error:
-- ROLLBACK;

DCL — Control de Acceso y Seguridad

El sublenguaje DCL (Data Control Language) se encarga de administrar los roles, usuarios y gestionar la seguridad.

-- 1. Crear un rol de base de datos
CREATE ROLE rol_analista;

-- 2. Asignar un usuario miembro a un rol (Sintaxis estándar / ANSI SQL)
GRANT rol_analista TO usuario_web;

-- Asignar miembro a un rol (Sintaxis específica de T-SQL / SQL Server)
ALTER ROLE rol_analista ADD MEMBER usuario_web;

-- 3. Gestión de Permisos: Conceder lectura
GRANT SELECT ON empleados TO rol_analista;

-- 4. Gestión de Permisos: Denegación explícita (bloquea el acceso aunque pertenezca a otros roles)
DENY INSERT, DELETE ON empleados TO rol_analista;

-- 5. Revocar permisos previamente concedidos
REVOKE SELECT ON empleados FROM rol_analista;
