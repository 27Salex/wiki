XML Schema Definition (XSD)

Validación avanzada de XML — Tipos de datos y Restricciones

¿Qué es XSD?

XSD (XML Schema Definition) es la alternativa moderna y potente a los DTD. Describe la estructura de un documento XML, pero a diferencia del DTD, un esquema XSD está escrito en sintaxis XML e incorpora un sólido sistema de tipos de datos y namespaces.

Ventajas de XSD sobre DTD:
  • Está escrito en XML (no requiere aprender otra sintaxis).
  • Soporta tipos de datos (enteros, fechas, booleanos, etc.).
  • Permite crear tipos de datos personalizados.
  • Soporta namespaces para evitar colisiones de nombres.
  • Permite aplicar restricciones (facets) muy detalladas a los datos.

Estructura Básica de un XSD

Todo documento XSD tiene como elemento raíz <xs:schema>.

<?xml version="1.0" encoding="UTF-8"?>
<xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema">

    <!-- Definición del elemento raíz -->
    <xs:element name="nota">
        <xs:complexType>
            <xs:sequence>
                <xs:element name="para" type="xs:string"/>
                <xs:element name="de" type="xs:string"/>
                <xs:element name="titulo" type="xs:string"/>
                <xs:element name="cuerpo" type="xs:string"/>
            </xs:sequence>
        </xs:complexType>
    </xs:element>

</xs:schema>

Tipos Simples vs Tipos Complejos

Tipos Simples (Simple Type)

Un elemento de tipo simple solo contiene texto. No puede contener otros elementos ni tener atributos.

<xs:element name="edad" type="xs:integer"/>
<xs:element name="precio" type="xs:decimal"/>
<xs:element name="fechaNacimiento" type="xs:date"/>

Tipos Complejos (Complex Type)

Un elemento de tipo complejo contiene otros elementos y/o atributos.

<!-- Elemento que contiene otros elementos -->
<xs:element name="empleado">
    <xs:complexType>
        <xs:sequence>
            <xs:element name="nombre" type="xs:string"/>
            <xs:element name="salario" type="xs:decimal"/>
        </xs:sequence>
        <!-- Atributo del elemento empleado -->
        <xs:attribute name="id" type="xs:string" use="required"/>
    </xs:complexType>
</xs:element>

Indicadores (Order, Occurrence, Group)

Indicadores de Orden

<!-- sequence: los elementos deben aparecer en este orden exacto -->
<xs:sequence>
    <xs:element name="nombre" type="xs:string"/>
    <xs:element name="apellidos" type="xs:string"/>
</xs:sequence>

<!-- all: los elementos pueden aparecer en cualquier orden, pero solo una vez -->
<xs:all>
    <xs:element name="telefono" type="xs:string"/>
    <xs:element name="email" type="xs:string"/>
</xs:all>

<!-- choice: solo uno de los elementos puede aparecer -->
<xs:choice>
    <xs:element name="tarjeta" type="xs:string"/>
    <xs:element name="transferencia" type="xs:string"/>
</xs:choice>

Indicadores de Ocurrencia (minOccurs, maxOccurs)

Equivalen a los cuantificadores ?, *, + de DTD.

<!-- Opcional (min 0, max 1) equivalente a ? -->
<xs:element name="segundoNombre" type="xs:string" minOccurs="0"/>

<!-- Al menos uno (min 1, max ilimitado) equivalente a + -->
<xs:element name="telefono" type="xs:string" minOccurs="1" maxOccurs="unbounded"/>

<!-- Cero o más (min 0, max ilimitado) equivalente a * -->
<xs:element name="hijo" type="xs:string" minOccurs="0" maxOccurs="unbounded"/>

<!-- Exactamente 3 -->
<xs:element name="intento" type="xs:string" minOccurs="3" maxOccurs="3"/>

Restricciones (Facets)

Permiten definir validaciones precisas sobre los datos (longitud, rangos, expresiones regulares, etc.).

Rango de valores (minInclusive, maxInclusive)

<xs:simpleType name="edadValida">
    <xs:restriction base="xs:integer">
        <xs:minInclusive value="18"/>
        <xs:maxInclusive value="120"/>
    </xs:restriction>
</xs:simpleType>

Lista de valores (Enumeration)

<xs:simpleType name="tallaRopa">
    <xs:restriction base="xs:string">
        <xs:enumeration value="S"/>
        <xs:enumeration value="M"/>
        <xs:enumeration value="L"/>
        <xs:enumeration value="XL"/>
    </xs:restriction>
</xs:simpleType>

Patrones (Expresiones Regulares)

<!-- Validar un código postal de 5 dígitos -->
<xs:simpleType name="codigoPostal">
    <xs:restriction base="xs:string">
        <xs:pattern value="[0-9]{5}"/>
    </xs:restriction>
</xs:simpleType>

Longitud de cadenas (Length)

<xs:simpleType name="password">
    <xs:restriction base="xs:string">
        <xs:minLength value="8"/>
        <xs:maxLength value="20"/>
    </xs:restriction>
</xs:simpleType>

Ejemplo Completo (XML y XSD)

Archivo XSD (empresa.xsd)

<?xml version="1.0" encoding="UTF-8"?>
<xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema">

    <!-- Definición de tipos simples -->
    <xs:simpleType name="tipoDNI">
        <xs:restriction base="xs:string">
            <xs:pattern value="[0-9]{8}[A-Z]"/>
        </xs:restriction>
    </xs:simpleType>

    <!-- Definición del elemento raíz -->
    <xs:element name="empresa">
        <xs:complexType>
            <xs:sequence>
                <xs:element name="empleado" maxOccurs="unbounded">
                    <xs:complexType>
                        <xs:sequence>
                            <xs:element name="nombre" type="xs:string"/>
                            <xs:element name="apellidos" type="xs:string"/>
                            <xs:element name="dni" type="tipoDNI"/>
                            <xs:element name="fecha_alta" type="xs:date"/>
                        </xs:sequence>
                        <xs:attribute name="id" type="xs:positiveInteger" use="required"/>
                    </xs:complexType>
                </xs:element>
            </xs:sequence>
        </xs:complexType>
    </xs:element>

</xs:schema>

Archivo XML (datos.xml)

Así se asocia el archivo XML con su esquema XSD para validación.

<?xml version="1.0" encoding="UTF-8"?>
<empresa xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:noNamespaceSchemaLocation="empresa.xsd">

    <empleado id="1">
        <nombre>Juan</nombre>
        <apellidos>Pérez Gómez</apellidos>
        <dni>12345678A</dni>
        <fecha_alta>2026-05-18</fecha_alta>
    </empleado>

    <empleado id="2">
        <nombre>María</nombre>
        <apellidos>López Ruiz</apellidos>
        <dni>87654321B</dni>
        <fecha_alta>2025-10-01</fecha_alta>
    </empleado>

</empresa>

Declaración de Atributos

Los atributos se declaran dentro de un complexType con <xs:attribute>.

<xs:element name="producto">
    <xs:complexType>
        <xs:sequence>
            <xs:element name="nombre" type="xs:string"/>
        </xs:sequence>

        <!-- Atributo obligatorio -->
        <xs:attribute name="codigo" type="xs:string" use="required"/>

        <!-- Atributo opcional (por defecto) -->
        <xs:attribute name="color" type="xs:string" use="optional"/>

        <!-- Atributo con valor por defecto -->
        <xs:attribute name="moneda" type="xs:string" default="EUR"/>

        <!-- Atributo con valor fijo (no se puede cambiar) -->
        <xs:attribute name="version" type="xs:string" fixed="2.0"/>

        <!-- Atributo con tipo personalizado -->
        <xs:attribute name="talla" type="tallaRopa"/>
    </xs:complexType>
</xs:element>

Extensión y Restricción de Tipos Complejos

XSD permite crear tipos que heredan de otros, ya sea añadiendo campos (extensión) o limitándolos (restricción).

Extensión (extension) — Herencia

<!-- Tipo base -->
<xs:complexType name="personaType">
    <xs:sequence>
        <xs:element name="nombre" type="xs:string"/>
        <xs:element name="email" type="xs:string"/>
    </xs:sequence>
</xs:complexType>

<!-- Tipo extendido: hereda de personaType y añade campos -->
<xs:complexType name="empleadoType">
    <xs:complexContent>
        <xs:extension base="personaType">
            <xs:sequence>
                <xs:element name="salario" type="xs:decimal"/>
                <xs:element name="departamento" type="xs:string"/>
            </xs:sequence>
            <xs:attribute name="id" type="xs:positiveInteger"/>
        </xs:extension>
    </xs:complexContent>
</xs:complexType>

<!-- Uso: empleado tendrá nombre, email, salario, departamento + id -->
<xs:element name="empleado" type="empleadoType"/>

Restricción de Tipo Simple (simpleContent + restriction)

<!-- Elemento con texto simple + atributo (simpleContent) -->
<xs:element name="precio">
    <xs:complexType>
        <xs:simpleContent>
            <xs:extension base="xs:decimal">
                <xs:attribute name="moneda" type="xs:string" use="required"/>
            </xs:extension>
        </xs:simpleContent>
    </xs:complexType>
</xs:element>
<!-- XML válido: <precio moneda="EUR">29.99</precio> -->

Contenido Mixto (Mixed Content)

Permite que un elemento contenga tanto texto libre como elementos hijos entremezclados.

<xs:element name="parrafo">
    <xs:complexType mixed="true">
        <xs:sequence>
            <xs:element name="negrita" type="xs:string" minOccurs="0" maxOccurs="unbounded"/>
            <xs:element name="cursiva" type="xs:string" minOccurs="0" maxOccurs="unbounded"/>
        </xs:sequence>
    </xs:complexType>
</xs:element>

<!-- XML válido: -->
<!-- <parrafo>Esto es <negrita>importante</negrita> y esto es <cursiva>énfasis</cursiva>.</parrafo> -->

Elementos Vacíos

Un elemento que no tiene contenido (ni texto ni hijos), pero puede tener atributos.

<xs:element name="imagen">
    <xs:complexType>
        <!-- Sin xs:sequence ni contenido = elemento vacío -->
        <xs:attribute name="src" type="xs:anyURI" use="required"/>
        <xs:attribute name="alt" type="xs:string" use="required"/>
        <xs:attribute name="ancho" type="xs:positiveInteger"/>
    </xs:complexType>
</xs:element>
<!-- XML: <imagen src="foto.jpg" alt="Logo" ancho="200"/> -->

Grupos Reutilizables

group — Grupo de elementos

<!-- Definir un grupo reutilizable -->
<xs:group name="datosContacto">
    <xs:sequence>
        <xs:element name="telefono" type="xs:string" minOccurs="0"/>
        <xs:element name="email" type="xs:string"/>
        <xs:element name="web" type="xs:anyURI" minOccurs="0"/>
    </xs:sequence>
</xs:group>

<!-- Reutilizar en múltiples tipos complejos -->
<xs:complexType name="clienteType">
    <xs:sequence>
        <xs:element name="nombre" type="xs:string"/>
        <xs:group ref="datosContacto"/>
    </xs:sequence>
</xs:complexType>

<xs:complexType name="proveedorType">
    <xs:sequence>
        <xs:element name="razon_social" type="xs:string"/>
        <xs:group ref="datosContacto"/>
    </xs:sequence>
</xs:complexType>

attributeGroup — Grupo de atributos

<xs:attributeGroup name="metadatos">
    <xs:attribute name="creado_por" type="xs:string"/>
    <xs:attribute name="fecha_creacion" type="xs:date"/>
    <xs:attribute name="version" type="xs:positiveInteger" default="1"/>
</xs:attributeGroup>

<!-- Reutilizar -->
<xs:complexType name="documentoType">
    <xs:sequence>
        <xs:element name="titulo" type="xs:string"/>
    </xs:sequence>
    <xs:attributeGroup ref="metadatos"/>
</xs:complexType>

Namespaces en XSD

Los namespaces evitan conflictos de nombres cuando se combinan múltiples esquemas.

<!-- XSD con targetNamespace -->
<?xml version="1.0" encoding="UTF-8"?>
<xs:schema
    xmlns:xs="http://www.w3.org/2001/XMLSchema"
    targetNamespace="http://www.miempresa.com/esquema"
    xmlns:emp="http://www.miempresa.com/esquema"
    elementFormDefault="qualified">

    <xs:element name="empresa">
        <xs:complexType>
            <xs:sequence>
                <xs:element name="nombre" type="xs:string"/>
            </xs:sequence>
        </xs:complexType>
    </xs:element>
</xs:schema>

<!-- XML que usa el namespace -->
<emp:empresa
    xmlns:emp="http://www.miempresa.com/esquema"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://www.miempresa.com/esquema empresa.xsd">
    <emp:nombre>TechCorp</emp:nombre>
</emp:empresa>
