MongoDB & NoSQL

Base de datos documental — CRUD, Aggregation, Tipos NoSQL

Principios de las Bases de Datos NoSQL

Las bases de datos NoSQL (Not Only SQL) surgen como alternativa a las bases de datos relacionales para manejar grandes volúmenes de datos, alta disponibilidad y esquemas flexibles.

Teorema CAP

Un sistema distribuido solo puede garantizar dos de tres propiedades simultáneamente:

  • C (Consistency): Todos los nodos ven los mismos datos al mismo tiempo.
  • A (Availability): Cada petición recibe una respuesta, aunque no sea la más reciente.
  • P (Partition Tolerance): El sistema sigue funcionando aunque haya fallos de red entre nodos.

Principios BASE (vs ACID)

ACID (Relacional)BASE (NoSQL)
AtomicityBasically Available
ConsistencySoft state
IsolationEventual consistency
Durability—

BASE prioriza la disponibilidad y acepta que los datos pueden no ser consistentes de forma inmediata, pero lo serán eventualmente.

Tipos de Bases de Datos NoSQL

1. Documental (Document Store)

Almacenan datos como documentos JSON/BSON. Cada documento puede tener una estructura diferente.

Ejemplos: MongoDB, CouchDB, Amazon DocumentDB
Uso ideal: Catálogos de productos, gestión de contenido, perfiles de usuario.

2. Clave-Valor (Key-Value Store)

El modelo más simple: cada dato se almacena como un par clave → valor.

Ejemplos: Redis, Amazon DynamoDB, Memcached
Uso ideal: Caché, sesiones de usuario, contadores en tiempo real.

3. Columnar (Wide-Column Store)

Organizan datos en familias de columnas. Cada fila puede tener columnas diferentes.

Ejemplos: Apache Cassandra, HBase, Google Bigtable
Uso ideal: Datos de series temporales, analítica a gran escala, IoT.

4. Grafos (Graph Database)

Almacenan datos como nodos (entidades) y aristas (relaciones). Optimizadas para recorrer relaciones complejas.

Ejemplos: Neo4j, Amazon Neptune, ArangoDB
Uso ideal: Redes sociales, detección de fraudes, motores de recomendación.

¿Qué es MongoDB?

MongoDB es una base de datos NoSQL documental de código abierto. Almacena datos en documentos BSON (Binary JSON) dentro de colecciones (equivalentes a tablas). No requiere esquema fijo.

Conceptos clave:
  • Database → Contenedor de colecciones
  • Collection → Grupo de documentos (≈ tabla)
  • Document → Registro individual en BSON (≈ fila)
  • Field → Par clave-valor dentro de un documento (≈ columna)
  • _id → Clave primaria autogenerada (ObjectId)

Operaciones CRUD

CREATE — Insertar documentos

// Insertar un documento
db.empleados.insertOne({
    nombre: "Ana García",
    email: "ana@empresa.com",
    salario: 35000,
    departamento: "IT",
    habilidades: ["JavaScript", "Python", "MongoDB"],
    direccion: {
        calle: "Gran Vía 42",
        ciudad: "Madrid",
        cp: "28013"
    },
    fechaAlta: new Date("2025-01-15"),
    activo: true
});

// Insertar múltiples documentos
db.empleados.insertMany([
    { nombre: "Carlos López", salario: 42000, departamento: "Ventas" },
    { nombre: "María Ruiz",   salario: 38000, departamento: "IT" }
]);

READ — Consultar documentos

// Buscar todos
db.empleados.find();

// Buscar con filtro
db.empleados.find({ departamento: "IT" });

// Buscar uno solo
db.empleados.findOne({ email: "ana@empresa.com" });

// Proyección: mostrar solo ciertos campos
db.empleados.find(
    { departamento: "IT" },
    { nombre: 1, salario: 1, _id: 0 }  // 1 = incluir, 0 = excluir
);

// Operadores de comparación
db.empleados.find({ salario: { $gt: 30000 } });    // mayor que
db.empleados.find({ salario: { $gte: 30000 } });   // mayor o igual
db.empleados.find({ salario: { $lt: 50000 } });    // menor que
db.empleados.find({ salario: { $lte: 50000 } });   // menor o igual
db.empleados.find({ salario: { $ne: 40000 } });    // distinto de
db.empleados.find({ departamento: { $in: ["IT", "Ventas"] } });

// Operadores lógicos
db.empleados.find({
    $and: [
        { salario: { $gte: 30000 } },
        { departamento: "IT" }
    ]
});

db.empleados.find({
    $or: [
        { departamento: "IT" },
        { salario: { $gt: 50000 } }
    ]
});

// Ordenar, limitar, saltar
db.empleados.find()
    .sort({ salario: -1 })  // -1 = descendente, 1 = ascendente
    .skip(10)               // saltar los primeros 10
    .limit(5);              // tomar solo 5

// Búsqueda en documentos anidados
db.empleados.find({ "direccion.ciudad": "Madrid" });

// Búsqueda en arrays
db.empleados.find({ habilidades: "Python" });          // contiene "Python"
db.empleados.find({ habilidades: { $all: ["Python", "MongoDB"] } }); // contiene ambos
db.empleados.find({ habilidades: { $size: 3 } });     // array de tamaño 3

UPDATE — Actualizar documentos

// Actualizar un campo (usa $set)
db.empleados.updateOne(
    { email: "ana@empresa.com" },
    { $set: { salario: 38000, departamento: "Data" } }
);

// Incrementar un valor numérico
db.empleados.updateOne(
    { email: "ana@empresa.com" },
    { $inc: { salario: 3000 } }  // suma 3000 al salario actual
);

// Actualizar múltiples documentos
db.empleados.updateMany(
    { departamento: "IT" },
    { $set: { bono: 2000 } }
);

// Añadir elemento a un array
db.empleados.updateOne(
    { email: "ana@empresa.com" },
    { $push: { habilidades: "TypeScript" } }
);

// Eliminar un campo
db.empleados.updateOne(
    { email: "ana@empresa.com" },
    { $unset: { bono: "" } }
);

// Reemplazar documento completo
db.empleados.replaceOne(
    { email: "ana@empresa.com" },
    { nombre: "Ana García", email: "ana@empresa.com", salario: 40000 }
);

DELETE — Eliminar documentos

// Eliminar uno
db.empleados.deleteOne({ email: "ana@empresa.com" });

// Eliminar múltiples
db.empleados.deleteMany({ activo: false });

// Eliminar TODOS los documentos de la colección
db.empleados.deleteMany({});

// Eliminar la colección entera (estructura incluida)
db.empleados.drop();

Aggregation Pipeline

El pipeline de agregación procesa documentos a través de una serie de etapas (stages), donde cada etapa transforma los datos.

db.empleados.aggregate([
    // 1. Filtrar: solo activos del departamento IT
    { $match: {
        activo: true,
        departamento: "IT"
    }},

    // 2. Agrupar: por departamento, calcular estadísticas
    { $group: {
        _id: "$departamento",
        totalEmpleados: { $sum: 1 },
        salarioMedio:   { $avg: "$salario" },
        salarioMax:     { $max: "$salario" },
        salarioMin:     { $min: "$salario" },
        nombres:        { $push: "$nombre" }
    }},

    // 3. Proyectar: dar formato a la salida
    { $project: {
        departamento: "$_id",
        totalEmpleados: 1,
        salarioMedio: { $round: ["$salarioMedio", 2] },
        _id: 0
    }},

    // 4. Ordenar: por salario medio descendente
    { $sort: { salarioMedio: -1 } },

    // 5. Limitar resultados
    { $limit: 10 }
]);

$lookup — JOIN entre colecciones

// Equivalente a LEFT JOIN en SQL
db.empleados.aggregate([
    { $lookup: {
        from: "departamentos",       // colección a unir
        localField: "depto_id",       // campo en empleados
        foreignField: "_id",          // campo en departamentos
        as: "info_departamento"       // nombre del array resultante
    }},
    { $unwind: "$info_departamento" } // desaplanar el array
]);

Índices en MongoDB

// Índice simple (ascendente)
db.empleados.createIndex({ email: 1 });

// Índice único
db.empleados.createIndex({ email: 1 }, { unique: true });

// Índice compuesto
db.empleados.createIndex({ departamento: 1, salario: -1 });

// Índice de texto (para búsquedas full-text)
db.articulos.createIndex({ titulo: "text", contenido: "text" });
db.articulos.find({ $text: { $search: "mongodb tutorial" } });

// Ver todos los índices
db.empleados.getIndexes();

// Eliminar un índice
db.empleados.dropIndex("email_1");
